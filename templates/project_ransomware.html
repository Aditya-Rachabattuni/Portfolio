Ransomware Classification and Detection

This project focuses on developing a machine learning model to detect ransomware by classifying executable files based on their attributes. Using Decision Tree and Random Forest algorithms, I built a robust system capable of accurately classifying files as either safe or infected with ransomware.

Technologies Used

  • Python: Core programming language used for machine learning model development.
  • Django: Web framework used to build the user interface and manage backend operations.
  • SQLite: Relational database used for storing uploaded files and classification results.
  • HTML, CSS, JavaScript: Frontend technologies used to design and implement the user interface for file uploads and results display.

Project Overview

This project utilizes machine learning techniques to detect ransomware by analyzing executable files. The model is trained on various features such as file size, subsystem version, entry point, and other metadata extracted from executable files. The trained machine learning model is then deployed in a Django-based web application, which allows users to upload files and receive real-time classification results. The system classifies files as either safe or ransomware-infected.

Data Collection & Preprocessing

The dataset for this project was curated by collecting both benign and ransomware-infected executable files. The files were analyzed to extract various features that can help differentiate between malicious and safe files. Some of the features extracted include:

  • File Size: The size of the executable file is one of the distinguishing features between malicious and benign files.
  • Subsystem Version: The subsystem version provides information on the compatibility of the executable file and can give insights into its legitimacy.
  • File Attributes: Various other file attributes such as timestamp, number of sections, and entry point are also used to determine if a file is malicious.

The data preprocessing steps included feature scaling, handling missing data, and encoding categorical variables, ensuring that the dataset is ready for model training.

Model Development

The classification model was developed using two machine learning algorithms:

  • Decision Tree Classifier: A tree-based algorithm that splits the data into branches based on feature values. Itâ€™s simple to understand and interpret, making it an ideal choice for this classification problem.
  • Random Forest Classifier: An ensemble method that combines multiple decision trees to improve classification accuracy and reduce overfitting. This model generally performs better on real-world data due to its ability to handle complexity and variance in data.

The models were trained using the features extracted from the dataset and validated through cross-validation to assess performance. After training, the models were evaluated on several metrics, including accuracy, precision, recall, and F1-score, to ensure reliable results.

Web Application Interface

The project also includes a user-friendly web application that allows users to upload executable files for classification. The Django framework was used to build the application, and the following features were incorporated:

  • File Upload: Users can upload executable files through a simple form on the website. The file is then processed for classification by the trained machine learning model.
  • Real-Time Classification: After file upload, the system immediately processes the file, classifies it, and displays the result (either "Safe" or "Ransomware Infected").
  • Result Display: The result is shown on the web page, indicating whether the uploaded file is safe or infected. Users can also view additional details about the classification process.
  • Backend Database (SQLite): The uploaded files and classification results are stored in an SQLite database, allowing for record-keeping and retrieval of past classifications.

Key Features

  • Real-time file analysis for classification using machine learning algorithms (Decision Tree and Random Forest).
  • User-friendly web application interface built with Django for easy file upload and results display.
  • Secure backend using SQLite for storing uploaded files and classification data.
  • Interactive front-end with HTML, CSS, and JavaScript for a seamless user experience.
  • Model performance evaluation based on metrics such as accuracy, precision, and recall.
  • Easy deployment and scalable architecture for future extensions and improvements.

Challenges & Future Work

During the project, several challenges were encountered, including handling large file sizes, ensuring the accuracy of the classification models, and addressing imbalanced datasets. In the future, I plan to:

  • Improve the dataset by incorporating more diverse file samples from different sources.
  • Enhance the user interface to allow for additional features, such as the ability to view the file metadata or submit multiple files at once.
  • Experiment with more advanced machine learning algorithms such as Support Vector Machines and XGBoost for potentially better accuracy and performance.
