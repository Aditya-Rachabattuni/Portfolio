Spam Filtering using Support Vector Machine

This project uses a Support Vector Machine (SVM) classifier to distinguish between spam and legitimate messages based on text features. The model is trained on a dataset containing labeled examples of spam and non-spam messages, which allows it to generalize and accurately classify new messages.

Technologies Used

  • Python: The primary programming language for model development, data preprocessing, and integration with the web app.
  • Streamlit: A Python library used to build the interactive web interface, which allows users to input text and receive real-time spam classification results.
  • Scikit-learn: The machine learning library used for implementing the SVM classifier, along with preprocessing steps like text vectorization and data scaling.
  • Natural Language Processing (NLP): Various NLP techniques were used to transform text data into a format suitable for machine learning models, such as TF-IDF (Term Frequency-Inverse Document Frequency).

Project Overview

The goal of this project was to create a real-time spam detection tool using Support Vector Machines (SVM). SVM is a powerful supervised learning algorithm that works well for binary classification tasks like distinguishing between spam and legitimate messages.

The model is trained using a dataset of messages, where each message is labeled as either "spam" or "not spam." The SVM classifier learns to distinguish between these classes by identifying patterns in the text features. Once trained, the model can be used to classify incoming messages in real-time.

The interactive interface for this tool was built using Streamlit, which allows users to input text and instantly see whether the message is classified as spam or legitimate.

Data Collection & Preprocessing

The dataset used in this project contains labeled text messages. Some key steps involved in preparing the data for model training include:

  • Text Preprocessing: Raw text data needs to be cleaned and transformed before it can be used to train machine learning models. This involves removing stop words, punctuation, and special characters, and converting all text to lowercase.
  • Tokenization: Tokenization is the process of splitting the text into individual words or tokens. These tokens serve as the features for the classification model.
  • TF-IDF Vectorization: We used the TF-IDF (Term Frequency-Inverse Document Frequency) technique to convert the text data into a numerical format. This method helps capture the importance of words based on their frequency in the message and across the entire dataset.
  • Label Encoding: The messages were labeled as "spam" or "not spam." These labels were encoded into numerical values (0 for "not spam" and 1 for "spam") to be used in the training process.

Model Development

The spam detection model was built using the Support Vector Machine (SVM) algorithm, which is well-suited for text classification tasks. The steps involved in developing the model are:

  • Training the SVM Classifier: We used Scikit-learn's SVC (Support Vector Classifier) implementation to train the model. The model is trained using the TF-IDF transformed text features as input and the encoded labels (spam or not) as output.
  • Hyperparameter Tuning: We tuned the SVM classifier’s hyperparameters (such as the regularization parameter C and the kernel type) using cross-validation and grid search to achieve the best possible accuracy.
  • Model Evaluation: The model's performance was evaluated using metrics such as accuracy, precision, recall, and F1-score. The confusion matrix was also used to visualize the number of true positives, false positives, true negatives, and false negatives.

After training, the model was able to classify text messages with high accuracy, ensuring that legitimate messages were not misclassified as spam, and vice versa.

Web Application Interface

The interactive web interface for the spam filtering tool was built using Streamlit, a Python library that allows for rapid development of machine learning web applications. The following features were incorporated into the application:

  • Text Input: Users can input a message in a text box, and the system will predict whether it is spam or not based on the model’s classification.
  • Real-Time Spam Classification: The input message is instantly classified as either "Spam" or "Not Spam," and the result is displayed on the screen.
  • Feedback System: After seeing the classification, users can submit feedback on whether the classification was correct, allowing for model improvement and evaluation.
  • Model Performance Metrics: Users can view model performance metrics such as accuracy, precision, recall, and F1-score directly on the interface, allowing them to evaluate the model’s effectiveness.

Key Features

  • Real-time spam classification using an SVM classifier trained on text features.
  • User-friendly and interactive interface built with Streamlit for easy text input and spam prediction.
  • Text preprocessing with TF-IDF vectorization and NLP techniques to extract meaningful features from the text data.
  • Model performance metrics displayed on the interface to help users understand the classifier's effectiveness.
  • Continuous learning feature, where users can provide feedback on the predictions to improve model accuracy over time.

Challenges & Future Work

While developing the project, some challenges were encountered, including:

  • Imbalanced Dataset: Spam datasets often contain more legitimate messages than spam, which can lead to biased predictions. Future work will involve using techniques like SMOTE (Synthetic Minority Over-sampling Technique) to balance the dataset and improve model performance.
